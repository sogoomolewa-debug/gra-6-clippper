### Actionable Recommendations for Automated Pipeline

To bridge the gap and match the premium quality of the template, focus on these areas:

#### 1. Caption Styling & Layout (FFmpeg `drawtext` filter)

*   **Font Choice:**
    *   **Recommendation:** Integrate a more robust and visually impactful font.
    *   **FFmpeg Action:** Download and include a bold, sans-serif font like **Oswald Bold**, **Impact**, **Montserrat Black**, or **Poppins Black** in your pipeline's font directory.
    *   **FFmpeg Filter:** Use `fontfile=/path/to/your/font.ttf` in your `drawtext` command.
*   **Font Size:**
    *   **Recommendation:** Increase the font size to create more visual dominance.
    *   **FFmpeg Action:** Experiment with `fontsize` values. Start with `fontsize=100` for 1080p vertical video and adjust.
*   **Text Color, Stroke, and Shadow:**
    *   **Recommendation:** Implement the thick black outline and a subtle drop shadow.
    *   **FFmpeg Action:**
        ```
        drawtext=fontcolor=white:borderw=6:bordercolor=black:shadowx=3:shadowy=3:...
        ```
        (Adjust `borderw`, `shadowx`, `shadowy` for desired thickness and depth)
*   **Capitalization:**
    *   **Recommendation:** Convert all hook text to ALL CAPS.
    *   **Pipeline Action:** Before passing text to FFmpeg, apply a `.upper()` method (or equivalent in your programming language) to the string.
*   **Positioning (Crucial for YouTube UI):**
    *   **Recommendation:** Ensure captions are perfectly centered for hooks and stay within YouTube Shorts UI safe zones for all captions.
    *   **FFmpeg Action:**
        *   **Centered (Hook):** `x=(w-text_w)/2:y=(h-text_h)/2`
        *   **Bottom-Centered (General):** `x=(w-text_w)/2:y=(h-text_h-120)` (The `-120` is a placeholder for a bottom margin of ~120 pixels; adjust based on actual testing to avoid UI overlap).
    *   **Pipeline Action:** Test thoroughly on a YouTube Shorts upload to confirm no UI overlap.
*   **Words per Screen (Pacing):**
    *   **Recommendation:** Break hook sentences into 1-2 word segments and display them sequentially with short durations.
    *   **Pipeline Action:**
        1.  Parse the hook sentence into an array of keywords/short phrases.
        2.  For each segment, calculate start and end timestamps.
        3.  Generate multiple `drawtext` filters in FFmpeg, each with its `enable='between(t,start_time,end_time)'` clause for precise timing.
        *Example (simplified):*
        ```
        -vf "drawtext=text='BIGGER':fontfile=...:x=...:y=...:enable='between(t,0,0.7)', \
             drawtext=text='FISH':fontfile=...:x=...:y=...:enable='between(t,0.7,1.4)'"
        ```

#### 2. Visual Framing & Blur (FFmpeg `crop`, `scale`, `gblur` filters)

*   **Blur Filter Properties:**
    *   **Recommendation:** Switch to a smooth Gaussian blur.
    *   **FFmpeg Action:** Replace your current blur method with `gblur`.
        ```
        -vf "gblur=sigma=10:steps=2:enable='between(t,0,hook_duration)'"
        ```
        (Experiment with `sigma` and `steps` values. `sigma` controls intensity, `steps` controls quality/smoothness.)
*   **Blur Transition:**
    *   **Recommendation:** Use an instant cut for the blur to lift.
    *   **FFmpeg Action:** Apply the `gblur` filter only for the exact duration of the hook. Do *not* use a fade-out effect for the blur itself.
*   **Gameplay Framing & Zoom:**
    *   **Recommendation:** Analyze the core action of the unblurred gameplay and crop/zoom to maximize its visibility within the 9:16 frame.
    *   **FFmpeg Action:**
        *   Use `crop` to extract a 9:16 segment from wider footage if necessary.
        *   Use `scale` and potentially `zoompan` if dynamic zooming is desired to focus on inventory or character.
        *   Example for cropping 16:9 to 9:16 and scaling to 1080p vertical:
            ```
            -vf "crop=ih*9/16:ih,scale=1080:1920"
            ```
            (Adjust `crop` and `scale` to intelligently frame the most important part of your game footage.)

#### 3. Audio Composition (Voice Generation, FFmpeg `amerge`, `volume` filters)

*   **TTS Tone Variation:**
    *   **Recommendation:** Upgrade your TTS engine or leverage advanced features of your current one for more natural and expressive speech.
    *   **Voice Generation Action:**
        *   **Explore advanced TTS providers:** Look into services like ElevenLabs, Google Cloud Text-to-Speech (especially Wavenet voices), or AWS Polly, which offer more natural-sounding voices and often support SSML.
        *   **Utilize SSML (Speech Synthesis Markup Language):** If your TTS provider supports it, use SSML tags to control pitch, rate, pauses, and emphasis (e.g., `<emphasis level="strong">cash</emphasis>`).
*   **Background Music vs. Sound Effects:**
    *   **Recommendation:** Remove continuous background music. Use a specific sound effect only for the hook.
    *   **Audio Composition Action:**
        *   **Hook SFX:** Find a suitable "whoosh" or "swell" sound effect.
        *   **FFmpeg Action (for SFX):** Overlay the hook SFX only during the hook period.
            ```
            -i input_gameplay.mp4 -i hook_sfx.wav \
            -filter_complex "[0:a]adelay=0|0[game_audio]; [1:a]adelay=start_time_sfx|start_time_sfx[sfx_audio]; \
                             [game_audio][sfx_audio]amerge=inputs=2[aout]" -map "[aout]" ...
            ```
            (Adjust `start_time_sfx` to align with your text reveal.)
        *   **Remove BG Music:** Ensure your pipeline *does not* add continuous background music. Prioritize raw game audio for gameplay segments.
*   **Volume Balance:**
    *   **Recommendation:** Ensure TTS (and hook SFX) is clearly audible during the hook, and game audio is clear during gameplay.
    *   **FFmpeg Action:** Use the `volume` filter to adjust levels if needed.
        ```
        -filter_complex "[game_audio]volume=0.8[game_vol]; [sfx_audio]volume=1.2[sfx_vol]; [game_vol][sfx_vol]amerge=inputs=2[aout]"
        ```
        (Adjust `volume` values based on source audio levels.)
